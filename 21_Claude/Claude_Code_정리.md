# Claude Code 학습 정리

Anthropic 공식 AI 코딩 CLI 도구. 터미널에서 자연어로 코드를 만들고 수정한다.

- 공식 문서: https://code.claude.com/docs/ko/

---

## 1. 설치

**요구사항**: Node.js 18 이상

```bash
npm install -g @anthropic-ai/claude-code

claude --version   # 설치 확인
claude             # 실행
```

---

## 2. 모델 선택

이름은 전부 **시(詩) 형식**에서 따왔다. 크기와 속도에 따른 3단계다.

| 모델 | 이름 유래 | 특징 |
|---|---|---|
| Haiku | 17음 일본 정형시, 세계에서 가장 짧은 시 | 가장 빠름, 간단한 작업 |
| Sonnet | 14행 유럽 정형시, 이탈리아어 "작은 노래" | 균형, 일반 작업 |
| Opus | 라틴어 "일, 작품", 예술의 대작 | 가장 강력, 복잡한 작업 |

**선택 기준**

- 빠른 응답 필요 -> Haiku
- 일반적인 작업 -> Sonnet
- 복잡한 분석, 코딩 -> Opus

```bash
claude --model sonnet   # CLI에서 모델 지정
```

대화 중에는 `/model`로 언제든 바꿀 수 있다.

## 3. 기초 명령어

| 명령어 | 설명 |
|---|---|
| `/init` | 프로젝트 설정 파일 CLAUDE.md 자동 생성 |
| `/help` | 모든 명령어 목록과 도움말 |
| `/clear` | 대화 히스토리 초기화 |
| `/compact` | 긴 대화를 요약, 압축해 컨텍스트 절약 |
| `/model` | 사용할 모델 변경 |
| `/usage` | 토큰 사용량과 비용 확인 |
| `/doctor` | 프로젝트 구조 분석과 문제 진단 |
| `/agents` | 사용 가능한 에이전트 확인과 생성 |
| `/rename` | 세션 이름 변경 |
| `/color` | 인터페이스 색상 테마 변경 |
| `/exit` | 세션 정상 종료 |

**자주 쓰는 것 정리**

- `/init` - 새 프로젝트 시작하면 가장 먼저 실행. 폴더 구조를 훑어 CLAUDE.md 초안을 만들어준다.
- `/clear` vs `/compact` - clear는 대화를 통째로 지운다(주제 전환용). compact는 지우지 않고 요약한다(작업 이어가며 절약).
- `/usage` - 토큰이 곧 비용이므로 긴 작업 중간에 점검한다.

## 4. 기호 명령

| 기호 | 기능 |
|---|---|
| `@파일명` | 특정 파일만 콕 집어 작업 대상 지정 |
| `!명령어` | 셸 명령 즉시 실행 (예: `!git status`) |
| 메시지 큐 | 작업 중에도 다음 명령 미리 입력 가능 |
| `/btw` | 메인 작업 중단 없이 간단한 질문. 컨텍스트에 안 남아 토큰 절약 |

`@파일명`은 엉뚱한 파일을 건드리는 사고를 막는 가장 확실한 방법이다.

## 5. 작동 모드

`Shift + Tab`으로 전환한다.

| 모드 | 동작 |
|---|---|
| default | 작업마다 사용자 확인을 받음 |
| auto | 확인 없이 모든 작업 자동 수행 |
| plan | 구현 전 계획서를 먼저 작성해 검토받음 |

### Plan 모드 (강력 추천)

1. **계획 먼저** - 코드 작성 전에 계획 문서(.md)를 만들어 보여준다.
2. **내가 검토** - 계획을 읽고 승인할지 결정한다. 수정 요청도 가능하다.
3. **승인 후 구현** - 승인해야 실제 코드 작성이 시작된다.

엉뚱한 방향으로 한참 간 뒤 처음부터 다시 하는 낭비를 막는다. 큰 작업일수록 효과가 크다.

## 6. 자율권과 안전

모드 선택은 결국 **AI에게 자율권을 얼마나 줄지** 정하는 일이다.

- 신입에게 처음엔 "매번 확인받고 진행"하다가, 신뢰가 쌓이면 "알아서 해"라고 맡기는 것과 같다.
- 다만 AI는 훨씬 빠르게 일한다. 잘못된 방향으로 갈 때도 그만큼 빠르다.
- **자율권 상승 = 속도 상승 = 위험 상승**. 셋은 함께 움직인다.

**판단 기준**: 작업을 **되돌릴 수 있는지**로 고른다.

- git으로 복구 가능한 코드 작성 -> 자동 모드
- 파일 삭제, 배포처럼 되돌리기 어려운 작업 -> 확인 모드

```bash
claude --dangerously-skip-permissions   # 모든 권한 확인 건너뛰기
```

옵션 이름에 `dangerously`(위험하게)가 대놓고 들어 있다. 신뢰할 수 있는 환경에서만 쓴다.

> "With great power comes great responsibility." (큰 힘에는 큰 책임이 따른다.)
> - Stan Lee, Spider-Man (Marvel Comics, 1962)

## 7. 하네스 엔지니어링

AI에게 자유를 주되 **위험한 순간에 제동을 거는** 기술이다.

- **하네스(Harness)** = 등산이나 고소 작업에서 쓰는 안전벨트.
- 안전줄을 매면 자유롭게 움직이되 추락은 막힌다. AI 에이전트에도 같은 원리를 적용한다.
- AI가 **오랜 시간 혼자 일할 때** 특히 중요하다. 방향을 잃거나 실수를 반복해도 사람이 계속 지켜볼 수 없기 때문이다.
- 방향을 잡아주고, 실수를 감지하고, 위험한 순간 자동으로 멈추는 장치를 미리 설계해둔다.
- Plan 모드, CLAUDE.md, 모드 선택이 전부 이 범주에 들어간다.

참고: Anthropic - Effective Harnesses for Long-Running Agents

## 8. CLAUDE.md 메모리 관리

프로젝트 규칙을 적어두는 파일. **세션 시작마다 자동으로 읽히므로** 매번 같은 설명을 반복하지 않아도 된다.

### 핵심 원칙: 200줄 이하 유지

- 모든 세션에서 로드되므로 길수록 컨텍스트를 많이 소비한다.
- 역설적으로 **파일이 길수록 규칙 준수율이 떨어진다**. 중요한 규칙이 묻히기 때문이다.
- 초과 내용은 `@import`나 `.claude/rules/`로 분리한다.

### 작성 노하우

| 팁 | 상세 |
|---|---|
| `/init`으로 시작 | 프로젝트 구조를 분석해 초안 자동 생성, 이후 직접 정제 |
| 포함할 것 | 빌드, 테스트, 린트 명령어, 코드 스타일, 브랜치 명명 규칙, 환경변수 주의사항 |
| 제외할 것 | 코드 보면 알 수 있는 것, 자주 바뀌는 정보, "깔끔한 코드 작성" 같은 당연한 말 |
| `@import` 문법 | 긴 문서는 외부 파일로 분리 (`@docs/git-instructions.md`) |
| 계층 구조 | 루트 CLAUDE.md(공통) + 하위 폴더 CLAUDE.md(모듈별) |
| 팀과 공유 | git에 커밋해 팀 전체가 일관된 컨텍스트 유지 |
| 개인 설정 분리 | `CLAUDE.local.md`는 `.gitignore`에 추가 |
| 강조 표현 | `IMPORTANT`, `YOU MUST`로 중요 규칙 준수율 향상 |
| 정기 정리 | Claude가 이미 잘 하는 것은 삭제. 남아있으면 비용 낭비 |

### 작성 예시 (3개 영역)

**글쓰기 규칙**
- 파일 연결 방식 통일 (구식 방법 혼용 금지)
- 이름 짓기 규칙 명시 (기능은 소문자, 화면은 대문자 시작)

**작업 순서**
- 수정 후 전체 오류 검사 필수
- DB 구조 파일 직접 수정 금지, 정해진 절차로만 변경

**Claude 당부**
- 작은 단계로 나눠서 진행
- 새 라이브러리 추가 전 먼저 확인

### 실전 예시

- **Andrej Karpathy의 CLAUDE.md** (65줄짜리 실전 예시)
  https://github.com/multica-ai/andrej-karpathy-skills/blob/main/CLAUDE.md
  테슬라 AI 총괄, OpenAI 창립 멤버. "200줄 이하" 원칙이 실제로 어떻게 지켜지는지 보여준다.

- 공식 문서: https://code.claude.com/docs/ko/memory

## 9. 스킬 (Skills)

자주 쓰는 작업을 파일로 저장해두고 **짧은 명령 하나로 실행**하는 기능. 요리 레시피를 저장해두는 것과 같다.

```
.claude/
  skills/
    code-review.md    -> /code-review
    translate-ko.md   -> /translate-ko
    daily-report.md   -> /daily-report
```

**스킬 파일 예시** (`translate-ko.md`)

```
주어진 텍스트를 자연스러운 한국어로 번역합니다
- 구어체보다 문어체 우선
- 전문 용어는 원문 병기
```

**사용 순서**

1. `.claude/skills/` 폴더 생성
2. `이름.md` 파일에 작업 지시 작성 (특별한 문법 없이 평범한 글)
3. Claude Code에서 `/이름` 입력하면 즉시 실행

## 10. 프롬프트 잘 쓰는 법

같은 요청도 **어떻게 쓰느냐**에 따라 결과가 크게 달라진다.

| 원칙 | 나쁜 예 | 좋은 예 |
|---|---|---|
| 위치 지정 | 버그 고쳐줘 | auth.js에서 비밀번호 찾기 버튼 오류 수정해줘 |
| 성공 기준 제시 | 이메일 확인 기능 만들어줘 | abc@test.com은 통과, 글자만 쓰면 오류 표시. 완성 후 직접 검사해줘 |
| 기존 방식 참조 | 새 위젯 추가해줘 | 기존 ProductCard 방식 그대로 달력 위젯도 만들어줘 |
| 금지사항 명시 | 코드 정리해줘 | 새 도구 추가 금지, 동작하는 기능은 건드리지 말 것 |
| 증상과 위치 함께 | 로그인이 안 돼 | 로그아웃 후 재로그인 시 실패. auth/ 에서 인증 갱신 확인해줘 |
| 이미지 활용 | 이 디자인처럼 만들어줘 | 화면 캡처 첨부 후 "이대로 만들고 결과도 캡처해 비교해줘" |
| 단계별 분해 | 로그인 기능 전체 만들어줘 | 현황 파악 -> 방법 설명 -> 코드 작성 순서로, 단계마다 확인 |
| 검증 수단 제공 | 코드 수정해줘 | 수정 후 자동 검사 돌려 기존 기능 문제없는지 확인해줘 |

### 프롬프트 구조화 4기법

| 기법 | 나쁜 방식 | 좋은 방식 |
|---|---|---|
| 주의력 분산 방지 | 1000줄 로그 전체 붙여넣기 | 관련 로그만 발췌해서 제시 |
| 역할 명확화 | 코드 좀 봐줘 | 당신은 시니어 백엔드 엔지니어. 성능 최적화 중심으로 |
| 단계별 분해 | 전체 시스템 다 만들어줘 | 1.분석 -> 2.문제 설명 -> 3.해결 방법 |
| 템플릿 활용 | 코드 왜 느려? | [ROLE] [GOAL] [CONSTRAINTS] [CONTEXT] [QUESTION] |

**실전 템플릿**

```
[ROLE: 시니어 엔지니어]
[GOAL: 버그 찾기]
[CONSTRAINTS: Python만]
[CONTEXT: 전체 코드]
[QUESTION: 정확한 질문]
```

### 질문 형식 활용

작업 요청 시 **먼저 질문하게** 시키면 원하는 결과를 정확히 얻는다.

```
로드맵 카드 디자인을 개선하고 싶어.
진행하기 전에 질문을 먼저 해줘.

질문 형식:
- 객관식 4개 + 주관식 1개
- 추천 답변에 별표 표시
- 다중 선택 가능
```

CLAUDE.md에 이 규칙을 등록하면 매번 요청하지 않아도 자동 적용된다.

## 11. Effort 레벨

AI의 **사고 깊이**를 조절한다. 높을수록 결과가 좋지만 토큰 소모와 비용도 올라간다.

| 레벨 | 적합한 작업 | 사고 깊이 |
|---|---|---|
| low | 간단한 질문, 오타 수정, 파일 이름 변경 | 얕고 빠름 |
| medium | 일반적인 코드 작성, 문서 정리 | 균형 |
| high | 복잡한 버그 분석, 아키텍처 설계, 리팩토링 | 깊고 느림 |
| xhigh | 초고난도 설계, 대규모 리팩토링, 심층 분석 | 매우 깊음 |
| max | 가장 깊은 사고, 토큰 제한 없음 | 최대 |

- Opus 기본값은 xhigh, Sonnet 기본값은 high다.
- `/model`로 모델을 바꿔도 effort는 **독립적으로** 조절 가능하다.

## 13. 기초 용어

**환경**

| 용어 | 설명 |
|---|---|

**AI 기초**

| 용어 | 설명 |
|---|---|

| Hallucination | AI가 사실이 아닌 내용을 자신 있게 생성하는 현상. 검증 필요 |

**프로젝트 설정**

**코드 관리**

| 용어 | 설명 |
|---|---|

**고급 용어**

| 용어 | 설명 |
|---|---|
| 프롬프트 엔지니어링 | 원하는 결과를 얻기 위해 질문과 지시를 잘 작성하는 기술 |
| 컨텍스트 엔지니어링 | AI 대화에 담을 정보를 전략적으로 구성하고 관리하는 기술 |

## AI 도입 여정

의미 있는 도구를 채택할 때 거치는 필연적 3단계다.

1. **비효율의 기간**
2. **적절성의 기간**
3. **워크플로와 삶을 변화시키는 발견의 기간**
