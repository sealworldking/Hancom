Q1CSS

position: absolute인 요소는 정확히 무엇을 기준으로 위치가 잡힐까?

HINT:
뭔가를 겹쳐서 띄우고 싶을 땐 부모한테 먼저 뭘 줘야 하는지, position 파트 다시 한번 떠올려보세요.

선택 안 함 답을 하나 먼저 골라주세요.

정답입니다
가장 가까운 조상 중 position이 static이 아닌(relative, absolute, fixed 등) 요소를 기준으로 배치됩니다.
그런 조상이 하나도 없으면 <html> 문서 전체(뷰포트) 기준으로 잡혀 의도와 다르게 튀어버립니다.
그래서 무언가를 겹쳐 띄울 땐 부모에 position: relative, 자식에 absolute를 주는 짝꿍 패턴을 항상 함께 씁니다.

아쉬워요, 정답은 ②번

HINT:
겉모습은 똑같아도 태그 이름 자체가 하는 역할이 따로 있어요. 그게 뭘지 생각해보세요.

Q4CSS

버튼 3개가 전부 같은 색을 쓰도록 :root에 --main-color 변수를 정의해 재사용했다. 변수 없이 색을 하드코딩했다면, 나중에 브랜드 색을 바꿀 때 무슨 문제가 생길까?

HINT:
:root 변수 하나만 고치면 한 번에 다 바뀌는데, 반대로 색을 다 따로따로 적어놨다면 나중에 얼마나 번거로울지 생각해보세요.

선택 안 함 답을 하나 먼저 골라주세요.

정답입니다
색이 쓰인 곳을 전부 찾아서 하나하나 값을 고쳐야 하고, 한 곳이라도 빠뜨리면 페이지 안에서 색이 미묘하게 어긋납니다.
:root 변수로 관리하면 --main-color 한 줄만 고쳐도 var(--main-color)를 쓰는 모든 곳이 한꺼번에 바뀌어 유지보수가 안전해집니다.

아쉬워요, 정답은 ②번
색이 쓰인 곳을 전부 찾아서 하나하나 값을 고쳐야 하고, 한 곳이라도 빠뜨리면 페이지 안에서 색이 미묘하게 어긋납니다.

Q5HTML, CSS

<meta name="viewport"> 태그 없이 만든 페이지를 폰으로 열면 화면이 어떻게 보일까?

HINT:
viewport가 없으면 브라우저는 화면 폭을 PC 기준으로 볼까요, 폰 기준으로 볼까요?

선택 안 함 답을 하나 먼저 골라주세요.

정답입니다
viewport 태그가 없으면 브라우저가 페이지를 PC 폭 그대로 렌더링한 뒤 폰 화면에 욱여넣어, 글자와 요소가 깨알같이 작게 보입니다.
viewport는 화면 폭을 기기 너비에 맞추는 반응형의 전제 조건이고, @media (max-width: ...)는 그 위에서 폭 조건별로 다른 스타일을 얹는 것입니다.
순서상 viewport가 먼저 있어야 media query도 의미가 있습니다.

아쉬워요, 정답은 ②번
viewport 태그가 없으면 브라우저가 페이지를 PC 폭 그대로 렌더링한 뒤 폰 화면에 욱여넣어, 글자와 요소가 깨알같이 작게 보입니다.
